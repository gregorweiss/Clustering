4 This software package provides extensive tools for robust and stable clustering of molecular dynamics trajectories.
5 The essential functions are:
6  - density-based geometric clustering for microstate generation
7  - dynamic clustering based on the Most-Probable-Path algorithm (MPP)
8  - variable dynamic coring for boundary corrections
9 
10 Additionally, the package includes tools to efficiently filter original coordinates or order parameters based on a discrete state definition
11 to identify representative structures and variables of clusters.
12 
13 Computationally demanding functions are parallelized in a hybrid model using OpenMP for SMP parallelization on a single node (multithreading)
14 and MPI over different cluster nodes. MPI support, however, is optional and for a modern computer with a high number of fast cores or
15 even multiple CPUs, OpenMP parallelization is sufficiently fast.

94 ## Optimized Binaries and Custom Build Options
95 
96 ### Vectorization
97 If you have a modern computer with vectorizing instruction sets (SSE2, SSE4_2, AVX, ...),
98 set the following **cmake-option**: -DDC_CPU_ACCELERATION=<OPTION>, where <OPTION> is one of
99  - SSE2
100  - SSE4_1
101  - SSE4_2
102  - AVX
103 
104 It is important to select an option that is actually supported by your machine.
105 Otherwise the program will produce erratic results, crash or not compile at all.
106 On linux systems, you can check your computer's capabilities with
107 
108  # cat /proc/cpuinfo
109 
110 Check in the *flags:*-block, if a certain instruction set is supported.
111 If it is not listed, it is not supported.
112 
113 ### Native Compilation
114 To compile the code with '-march=native' option (specific to the GNU compiler), add '-DDC_NATIVE_COMPILATION=ON' to your cmake-flags.
115 Using this option, the GNU compiler will automatically use all available instruction sets for optimal performance (**attention**: you still
116 need to set the vectorization option above, even if you use this option).
117 
118 Unfortunately, the resulting binary will most likely run only on the computer it was compiled on - do **not** use this option if you want
119 to distribute the binary, e.g. on a cluster.
120 
121 ### MPI
122 For MPI support, build your binary with the additional cmake-flag -DDC_USE_MPI=ON.
123 Invoke the *clustering_mpi* binary in the following way to run on several nodes with local multithreading via OpenMP:
124 
125  # /usr/bin/mpirun -n N_NODES -bind-to-core -bynode -cpus-per-proc N_THREADS_PER_NODE -report-bindings \
126  clustering_mpi density -f COORDS_FILE -r RADIUS -p POPS_OUT -d FE_OUT -n N_THREADS_PER_NODE
