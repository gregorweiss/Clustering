4 This software package provides extensive tools for fast, robust and stable
5 clustering of molecular dynamics trajectories.
6 The essential functions are:
7  - density-based geometric clustering for microstate generation
8  - dynamic clustering based on the Most-Probable-Path algorithm (MPP)
9  - variable dynamic coring for boundary corrections
10  - dynamic noise assignment.
11 
12 Additionally, the package includes tools to efficiently filter original
13 coordinates or order parameters based on a discrete state definition
14 to identify representative structures and variables of clusters.

91 ## Quick-Start
92 
93 To quickly get a working (but possibly underperforming) binary
94 
95  - unpack the code ...
96 ```
97  # tar xfz clustering_VERSION.tar.gz
98 ```
99  - create a build folder inside the code directory ...
100 ```
101  # cd clustering_VERSION
102  # mkdir build
103 ```
104  - change to the build directory ...
105 ```
106  # cd build
107 ```
108  - ... and run cmake
109 ```
110  # cmake .. -DCMAKE_INSTALL_PREFIX=/my/installation/path
111 ```
112  - then compile and install the package to /my/installation/path (or any other
113  path you chose above) by invoking
114 ```
115  # make
116  # make install
117 ```

119 ## Optimized Binaries and Custom Build Options
120 
121 ### CUDA
122 If you have an Nvidia graphic card it can be used to significantly speed up the
123 *clustering density* method, by setting the following **cmake-option**:
124 -DUSE_CUDA=1
125 
126 ### Vectorization
127 If you have a modern computer with vectorizing instruction sets (SSE2, SSE4_2,
128 AVX, ...), set the following **cmake-option**: -DCPU_ACCELERATION=<OPTION>,
129 where <OPTION> is one of
130  - SSE2
131  - SSE4_1
132  - SSE4_2
133  - AVX
134 
135 It is important to select an option that is actually supported by your machine.
136 Otherwise the program will produce erratic results, crash or not compile at all.
137 On linux systems, you can check your computer's capabilities with
138 ```
139  # cat /proc/cpuinfo
140 ```
141 Check in the *flags:*-block, if a certain instruction set is supported.
142 If it is not listed, it is not supported.
143 
144 ### Native Compilation
145 To compile the code with '-march=native' option (specific to the GNU compiler),
146 add '-DNATIVE_COMPILATION=ON' to your cmake-flags.
147 Using this option, the GNU compiler will automatically use all available
148 instruction sets for optimal performance (**attention**: you still
149 need to set the vectorization option above, even if you use this option).
150 
151 Unfortunately, the resulting binary will most likely run only on the computer
152 it was compiled on - do **not** use this option if you want
153 to distribute the binary, e.g. on a cluster.
